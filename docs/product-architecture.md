# 99小说 · 软件构造（优化版）

## 产品定位
- 目标：做一个“阅读 + 写作 + 自我洞察”一体化小说应用。
- 差异点：不只看书，还把阅读偏好与写作内容转化为人格画像与每日运势建议。

## 信息架构
- 书城模块
  - 分类筛选（科幻、悬疑、治愈、现实、奇幻、历史、成长、情感）
  - 搜索书名/作者/关键词
  - 书籍卡片 + 当前阅读台
- 创作模块
  - 标题输入 + 长文本编辑
  - 草稿本地保存（最近 6 条）
  - 字数与创作提示
- 人格模块
  - 基于 Big Five（开放性、责任感、外向表达、宜人性、情绪稳定）
  - 输出人格标签（如“共情造梦者”“结构创想家”）
  - 给出行为建议
- 每日运势模块
  - 基于“当日阅读行为 + 写作行为 + 用户种子”生成日签
  - 给出创作策略（开新坑、推进主线、修文打磨等）

## 交互流程
1. 用户完成 SecondMe 登录。
2. 回调后进入主页面 `/app`。
3. 用户选书阅读，阅读时长自动累计。
4. 用户在创作台写作并保存草稿。
5. 人格分析与运势实时刷新，形成“阅读-写作-洞察”闭环。

## 技术构成
- 后端：Node.js + Express + express-session
- 认证：SecondMe OAuth 授权码模式
- 页面：
  - 入口页 `/`（登录与状态提示）
  - 主页面 `/app`（完整小说应用 UI）
  - 数据接口 `/api/profile`（用户信息同步）
- 前端：
  - `public/app.css`（番茄风格参考下的暖色沉浸布局）
  - `public/app.js`（分类、阅读、写作、人格、运势交互逻辑）

## 设计原则
- 视觉风格：暖色系沉浸阅读氛围，强调“书感”和“情绪感”。
- 布局策略：左侧阅读主任务，右侧创作与人格洞察，兼顾移动端。
- 反馈机制：每个行为都有可见反馈（字数、阅读时长、画像变化、日签建议）。
